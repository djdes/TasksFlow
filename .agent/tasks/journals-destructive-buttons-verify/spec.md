# journals-destructive-buttons-verify

## Goal
Проверить на production реальные destructive-кнопки внутри новых документов каждого из 35 журналов:
- создать новый документ;
- заселить в него тестовые данные;
- реально выполнить удаление через UI;
- реально выполнить закрытие журнала через UI;
- при сбоях исправить код, перепроверить и только потом считать задачу завершенной.

## Scope
- Использовать `admin@haccp.local / admin1234`.
- Создавать отдельный новый документ для каждого кода из `.agent/tasks/journals-external-api/prod-journal-codes.txt`.
- Для заполнения тестовой строки/записи использовать production external API либо внутренние document API без удаления существующих пользовательских документов.
- Проверять именно UI-кнопки detail-страницы журнала, а не только прямые API-вызовы.
- Созданные этой проверкой документы можно оставить; удалять разрешено только строки/записи внутри них.

## Acceptance Criteria
- `AC1`: Для каждого из 35 журналов создан новый тестовый документ, и его `documentId` зафиксирован в evidence.
- `AC2`: В каждом созданном документе destructive-действие удаления действительно выполняется через UI и приводит к серверному изменению состояния документа.
- `AC3`: В каждом созданном документе кнопка `Закончить журнал` действительно выполняется через UI и переводит документ в `closed`.
- `AC4`: Если удаление в конкретном журнале требует отдельного `Сохранить`, verifier подтверждает persisted-изменение только после фактического сохранения.
- `AC5`: Артефакты записаны в `.agent/tasks/journals-destructive-buttons-verify/evidence.md`, `.agent/tasks/journals-destructive-buttons-verify/evidence.json`, raw-скриншоты и `problems.md`.
- `AC6`: Если хотя бы один журнал падает, completion не заявляется до минимального исправления, повторной проверки и `PASS` по всем `AC1-AC5`.
